Welcome to Bronto and Mozu

Bronto is the top-ranked email marketing platform to the Internet Retailer Top 1000. It features an advanced marketing and email automation engine designed for email, mobile, and social campaigns. With Bronto, you can handle shopping cart abandonment, post-purchase follow-ups, shopper segmentation, and more.

The Mozu Bronto Application integrates the benefits of Bronto with your Mozu account. After configuring and enabling the Mozu Bronto Application, you can access the functionality of the application using extensions that install in the Customers and Orders modules.

Application Features:

  • High level of granularity for choosing which marketing and transactional emails to send to shoppers, as well as what information to send between Bronto and Mozu regarding shoppers and orders
  • Sync the marketing preferences for your shoppers between your Bronto and Mozu accounts.
  • Export shopper and order information from Mozu to Bronto to take advantage of follow-up email campaigns.
  • Enable Bronto email campaigns in Mozu, such as order confirmations, wishlist reminders, order abandonment follow-ups, and in-stock notifications.
  • Map Mozu shopper attributes and order history to Bronto lists to organize your marketing campaigns.
  • Preview how emails sent to shoppers will look.
  • View logs related to shopper accounts and orders.
  • Use Mozu variables to customize email templates in Bronto.

Install the Application on Your Tenant

You can install the Bronto Application by Mozu directly from the Mozu App Marketplace:

  1. Go to: www.mozu.com/marketplace
  2. In the Search field, enter: Bronto
  3. Click the app icon.
  4. On the app page, click Install Now.

Configure the Application

To integrate your Bronto services with Mozu, review the configuration requirements to ensure you have everything you need to be successful before you begin, and then complete the steps in this section.

Configuration Requirements

  • The Mozu Bronto Application must be installed on your tenant.
  • You must have an active Bronto account.
  • You must have configured campaigns, customer lists, and email templates in your Bronto account.

Obtain Your Bronto API Key

The Bronto SOAP API Key grants Mozu access to your Bronto contacts and email templates without requiring you to enter your Bronto credentials in Mozu. Note this API key and enter it in Mozu during configuration:

  1. Log in to your Bronto account.
  2. Find the list of SOAP API Access Tokens for your account. At the time of this writing, you could find this list on the Data Exchange page.
  3. Note the key you want to use from the list. If you have not yet created a key, click Add Access Token to configure a new key with read, write, and send permissions for use with the Mozu Bronto Application.

Configure the Mozu Bronto Application

After obtaining your Bronto API key, enter the key in Mozu and configure the settings for the Mozu Bronto Application:

  1. Log in to Mozu Admin.
  2. On the Mozu Admin top bar, select Settings > Applications.
  3. Click Mozu Bronto Application.
  4. Click the Configuration link to open configuration settings.
  5. Open the Account tab.
  6. Enter your Bronto API Key.
  7. Click Save.

A Features tab displays.

  1. Open the Features tab.
  2. Choose which features to enable or disable for the Mozu Bronto Application. Many of the features require you to select a template for the emails sent to shoppers. If you want to add or edit these templates, you can do so from within your Bronto account.

    1. Enable Opt in/Out Sync from Mozu to Bronto—Sends shopper information from Mozu to Bronto about shoppers who opted into email marketing during the checkout process.

      If you do not enable this feature, Bronto classifies shoppers who receive non-transactional emails (wishlist reminder, welcome email, store credit) as "Onboarding Contacts", and classifies shoppers who receive transactional emails (in-stock notification, order confirmation, order shipment, RMA emails, password reset, and order abandonment) as "Transactional Contacts".

    2. Enable Opt in/Out Sync from Bronto to Mozu—Sends your Bronto contacts' marketing preferences from Bronto to Mozu.
    3. Enable Contact Export—Sends contact information from Mozu to Bronto about shoppers, including names, email addresses, and custom shopper attributes.
    4. Enable Welcome Emails—Sends a welcome email to new shoppers.
    5. Enable Password Reset Emails—Sends password reset emails to shoppers using a Bronto template instead of the default Mozu email.
    6. Turn on wishlist reminder emails—Sends a reminder email after a specified number of idle Day(s) to shoppers who have items in their wishlists.
    7. Enable Order Export from Mozu to Bronto—Sends order information from Mozu to Bronto. If you enable this feature, use the Contact Sync Settings tab to specify whether you want to place the shopper into a Bronto list:
      1. Open the Contact Sync Settings tab.
      2. Choose to enable or disable adding Mozu shoppers to lists in Bronto by clicking Add Contacts to List.
      3. Type an existing list name to add shoppers to, or type a new name to create a new list in Bronto to add shoppers to.
      4. Return to the Features tab.
    8. Enable Order Confirmation Email—Sends an order confirmation email to shoppers after the checkout process is complete.
    9. Enable Order Abandonment Emails—Sends an email to shoppers who began the checkout process 48 hours ago but who have not yet submitted the order.
    10. Enable Order Shipment Emails—Sends an email to shoppers when their order ships.
    11. Enable Return Created Emails—Sends an email to shoppers after they create a return.
    12. Enable Return Authorized Emails—Sends an email to shoppers after you receive and accept a return in Mozu using the Orders module.
    13. Enable Return Rejected Emails—Sends an email to shoppers after you reject a return in Mozu using the Orders module.
    14. Enable Return Closed Emails—Sends an email to shoppers after you close a return in Mozu using the Orders module.
    15. Enable InStock notification Emails—Sends an email to shoppers when an item on their wishlist is back in stock.
  3. Click Save.

Enable the App

After configuration, enable the Mozu Bronto Application in Mozu to apply its functionality to your tenant.

  1. On the Mozu Admin top bar, select Settings > Applications.
  2. Click Mozu Bronto Application.
  3. Click Enable App.

After enabling the Mozu Bronto Application, perform an initial export of shopper and order information from Mozu to Bronto.

Export Shopper Information to Bronto

Export Mozu shopper information using the Customers module extension. The export adds existing Mozu shoppers and their attributes to your Bronto contacts. After the first export, the Mozu Bronto Application automatically sends information about new Mozu shoppers to Bronto if you enable the contact export feature in the configuration settings.

  1. In Mozu Admin, click the Customers module.
  2. Click Extensions > Bronto > Attributes.
  3. Use the Attribute Mapping page to map shopper attributes gathered or created in Mozu to fields in your Bronto account. For example, you can map typical Mozu shopper attributes, like a shopper's first name, to a corresponding Bronto field. You can also map any custom shopper attributes you have created in Mozu to a Bronto field. If you don't select a field for an attribute, it will not map to Bronto.
  4. Click Save.
  5. Click Sync to open the Sync page.
  6. Click Export All Contacts To Bronto.
  7. Click Export to Bronto to send existing Mozu shopper information and marketing preferences to Bronto.

Export Order Information to Bronto

Export Mozu order information using the Orders module extension. The export sends order information from Mozu to Bronto about orders your site received prior to enabling the Mozu Bronto Application. After you enable the application, new order information is automatically sent to Bronto if you enable the order export feature in the configuration settings.

  1. In Mozu Admin, click the Orders module.
  2. Click Extensions > Bronto > Export.
  3. Click Export.

Congratulations! You can now preview Bronto functionality on your site.

Use the App

  • Customer Module Extension: Sync Contacts, Send Wishlist Reminders, View Logs, and Preview Emails
  • Orders Module Extension: View Logs and Preview Order & Return Emails
  • Use API Variables to Customize Email Templates

    • Customer Module Extension: Sync Contacts, Send Wishlist Reminders, View Logs, and Preview Emails

    The Mozu Bronto Application adds an extension to the Customers module that allows you to sync contact information between Mozu and Bronto, manage wishlist reminders, view related logs, and preview emails sent to shoppers. Note: The pages that display in this extension depend on the features you enable during configuration.

    To view this extension:

    1. In Mozu Admin, click the Customers module.
    2. Click Extensions > Bronto, and select a page.

    Sync Contacts between Mozu and Bronto

    Use the Sync page in the Bronto extension to update Mozu shoppers with their marketing preferences stored in Bronto. Any change in subscription settings in Bronto automatically update in Mozu.


    1. Click Sync Marketing Preferences From Bronto.
    2. Click Sync from Bronto to import shopper information and marketing preferences from Bronto.

    After you sync contacts between Mozu and Bronto, Mozu receives updates from Bronto once every hour.

    Run Wishlist Reminder

    Use the Wishlist page in the Bronto extension to remind shoppers about products on their wishlist that are available for purchase. Click Run Wishlist Reminder to send an on-demand email reminder. If you enabled wishlist reminders during configuration, a new reminder will run once per day at 12:00 AM CST to shoppers who have had items in their wishlist for the specified idle period.

    View Shopper Marketing Logs

    Use the Logs page in the Bronto extension to view events related to shopper marketing preferences. The log displays information about when shopper accounts were created, when marketing preferences were changed for an account, and when updates were made to shopper accounts. You can use the Search to filter the logs by shopper email.

    Send a Test Email

    Use the Email Test page in the Bronto extension to preview how an email will look to a shopper:


    1. Select an Email Type to preview from the drop-down menu.
    2. Enter an Email address associated with one of your shopper accounts. The information from the shopper account is used to populate the email with relevant data (for example, a shopper's name and wishlist).
    3. Click Test Email to send the test email to the email address associated with your Mozu account.

    Orders Module Extension: View Logs and Preview Order & Return Emails

    The Mozu Bronto Application adds an extension to the Orders module that allows you to view order export logs and preview emails sent to shoppers. Note: The pages that display in this extension depend on the features you enable during configuration.

    To view this extension:

    1. In Mozu Admin, click the Orders module.
    2. Click Extensions > Bronto, and select a page.

    View Order Logs

    Use the Logs page in the Bronto extension to view events related to order exports. You can use the Search to filter the logs by shopper email.

    Send a Test Email

    Use the Email Test page in the Bronto extension to preview how an email related to a specific order or return will look.


    1. Select an Email Type to preview from the drop-down menu.
    2. Enter an Order/Return Number. The information from the order or return is used to populate the email with relevant data (for example, the product ordered).
    3. Click Test Email to send the test email to the email address associated with your Mozu account.

Use API Variables to Customize Email Templates

Mozu exports an assortment of variables through the Mozu API related to shoppers and orders. You can use these variables to add useful content to your Bronto email templates. The syntax for using API variables in Bronto email templates is %%#variable%%. For example, if you are creating an email template to notify shoppers of a product that has come back into stock, you could enter the following text in the HTML for the Bronto In-Stock email template:

Good news! %%#productName%% is back in stock!

Also note that you must place email variables that end in _# within loops. For example:

{loop}%%#productImageUrl_#%%{/loop}
