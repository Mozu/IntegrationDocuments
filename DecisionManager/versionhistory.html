August, 2015

Thanks for using the Mozu CyberSource Decision Manager Application! Here's what new in this version:

  • You can now map Mozu fields (for example, the email address or payment type associated with an order) to MDD fields in Decision Manager. You can then associate the MDD data in Decision Manager with custom fields in order to create fraud detection rules that leverage the data mapped by Mozu.
  • You can now specify which payment types (credit card, PayPal Express, check, or store credit) to send to Decision Manager, allowing you to select all, none, or some combination of payment types to screen.
  • The Mozu CyberSource Decision Manager Application now supports Visa Checkout as a valid payment method in Mozu.
  • Pending orders are now included in the total order count within Decision Manager.
  • Performance improvements: Removed unnecessary calls to Decision Manager when you select CyberSource as your payment gateway in Mozu.
  • Bug fixes: Problems associated with sending credit card details to Decision Manager.

May, 2015

Initial release of the Mozu CyberSource Decision Manager Application! This application enables you to:

  • Automatically sends information about submitted orders to Decision Manager, including the shopper's browser type, IP address, device type, billing information, etc.
  • Validates shopper purchases against your Decision Manager rules to determine a fraud score and validation result (accept, review, decline).
  • Based on the validation result, changes the order status in Mozu to Accepted or Pending Review.
  • Supports syncing order status information with Decison Manager at scheduled intervals. If you enable syncing, the status of an order that is pending review automatically updates in Mozu if you accept or cancel the order through the Decision Manager website.
  • Set a monetary threshold below which orders are not screened for fraud to avoid excessive overhead for low-value orders.
