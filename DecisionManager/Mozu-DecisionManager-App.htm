Welcome to Decision Manager and Mozu

CyberSource's Decision Manager provides a fraud protection platform that features the World's Largest Fraud Detection Radar as well as a flexible rules engine that lets you customize rules to suit your business case. With Decision Manager, you can screen orders for risk in an effort to prevent fraud.

The Mozu CyberSource Decision Manager Application integrates your Decision Manager account with your site so that any order you receive on your site is automatically screened by Decision Manager. With the Mozu CyberSource Decision Manager Application, every order receives a fraud score and a validation result from Decision Manager. Based on these metrics, orders are either accepted in Mozu or marked as needing further review.

Application Features:

  • Automatically sends information about submitted orders to Decision Manager, including the shopper's browser type, IP address, device type, billing information, etc.
  • Validates shopper purchases against your Decision Manager rules to determine a fraud score and validation result (accept, review, decline).
  • Based on the validation result, changes the order status in Mozu to Accepted or Pending Review.
  • Supports syncing order status information with Decison Manager at scheduled intervals. If you enable syncing, the status of an order that is pending review automatically updates in Mozu if you accept or cancel the order through the Decision Manager website.
  • Set a monetary threshold below which orders are not screened for fraud to avoid excessive overhead for low-value orders.

Install the Mozu CyberSource Decision Manager Application on Your Tenant

After purchasing the Mozu CyberSource Decision Manager Application, call your sales or professional services representative and request installation on your tenant. Mozu staff will install the app to your tenant, making it available for further configuration and usage in your storefront.

Configure the Application

To integrate your Decision Manager services with Mozu, review the configuration requirements to ensure you have everything you need to be successful before you begin, and then complete the steps in this section.

Configuration Requirements

  • The Mozu CyberSource Decision Manager Application must be installed on your tenant.
  • You must have an active Decision Manager account.

Obtain Decision Manager Account Information

Note your Decision Manager account credentials. You will enter these credentials in Mozu.

  1. Log in to the CyberSource Business Center.
  2. Note the username and password that you use to log in to the CyberSource Business Center.
  3. Note your Account ID and Merchant ID.
  4. Create and note a Transaction Security Key for the SOAP Toolkit API.

    Note: If you have already created this key but lost or forgotten it, you will have to generate a new key.

Configure the Mozu CyberSource Decision Manager Application

After obtaining your Decision Manager account information, enter the information in Mozu and configure the settings for the Mozu CyberSource Decision Manager Application:

  1. Log in to Mozu Admin.
  2. On the Mozu Admin top bar, select Settings»Applications.
  3. Click CyberSource Decision Manager.
  4. Click the Configuration link to open configuration settings.
  5. Open the Settings tab.
  6. Add your CyberSource account information to Mozu:
    1. Enter your Organization ID.
    2. Enter your Merchant ID.
    3. Enter your Transaction Key.
    4. (Optional) Enable Threshold Enabled to set a Threshold Value below or at which orders are not screened by Decision Manager. For example, you may want Decision Manager to only screen orders over $10 for fraud.
    5. Choose which Environment the Mozu CyberSource Decision Manager Application screens orders in. If you select the Test environment, the application sends order information to Decision Manager's test site, where you can evaluate fraud detection rules in a sandbox setting. When you are satisfied with the rules you set in Decision Manager, choose the Production environment to enable live fraud detection on your site.
    6. Select an Order Synch Frequency to specify how often Mozu queries Decision Manager for updates on orders that are pending review. If you accept or reject an order that is pending review using the Decision Manager website instead of Mozu, then the order sync frequency determines how long it takes for the new order status to update in Mozu.

      Tip: Balance the frequency at which you receive updates in Mozu with the amount of bandwidth you are comfortable using to send order sync queries from Mozu to Decision Manager.

    7. If you specify an order synch frequency, enter the User Name and Password that you use to log into the CyberSource Business Center.
  7. (Optional) If you have more than one site, the settings you specify for the Mozu CyberSource Decision Manager Application apply to every site. However, if you want to configure different settings for a particular site, click Add Site (e.g., you want to use different Decision Manager accounts/transaction keys for a particular site). Any settings you configure in the new dialog apply specifically to the site you select in the drop-down.
  8. Click Save.

Enable the App

After configuration, enable the Mozu CyberSource Decision Manager Application in Mozu.

  1. On the Mozu Admin top bar, select Settings»Applications.
  2. Click CyberSource Decision Manager Application.
  3. Click Enable App.

Congratulations! You can now preview Decision Manager functionality on your site.

Handling Orders Flagged for Further Review

When Decision Manager screens orders it receives from Mozu, it attaches one of the following validation results to the order: accept, review, or reject. Orders tagged with "accept" change the order status in Mozu to Accepted. No further action is required for these orders from a fraud screening perspective. Orders tagged with "review" or "reject" change the order status in Mozu to Pending Review. To continue with the order process, accept or cancel the order in Mozu Admin:

  1. In Mozu Admin, open the Orders module.
  2. Open the Orders editor for an order that is pending review.
  3. Under Attributes, use the Decision Manager Fraud Results to review the reasons the order is pending review.
    1. Note whether Decision Manager accepted, rejected, or marked the order for further review.
    2. Consider fraud risk information such as the fraud score result (afsResult), risk factor codes (afsFactorCode), and reason code (ReasonCode). For help interpreting all the values listed in the Decision Manager results, refer to the Decision Manager User Guide available through the CyberSource Business Center.

    3. If you think the order is fraudulent, click Cancel Order. Otherwise, click Accept Order.

    You can also use the Decision Manager website to accept or cancel orders pending review. If you update an order through the Decision Manager website, you see the update in Mozu in accordance with the order synchronization interval you set in the configuration settings.

Customizing Fraud Detection Rules

If you want to change the rules that Decision Manager uses to screen an order for fraud risk, use the Decision Manager website to modify the existing rules or create new custom rules. CyberSource recommends making use of the first two merchant-defined data fields (MDD1 and MDD2) for custom testing. For more information on creating custom rules and fields in Decision Manager, refer to the Decision Manager User Guide available through the CyberSource Business Center.
