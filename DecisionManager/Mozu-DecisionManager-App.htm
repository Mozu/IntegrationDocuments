Welcome to Decision Manager and Mozu

CyberSource's Decision Manager provides a fraud protection platform that features the World's Largest Fraud Detection Radar as well as a flexible rules engine that lets you customize rules to suit your business case. With Decision Manager, you can screen orders for risk in an effort to prevent fraud.

The Mozu CyberSource Decision Manager Application integrates your Decision Manager account with your site so that any order you receive on your site is automatically screened by Decision Manager. With the Mozu CyberSource Decision Manager Application, every order receives a fraud score and a validation result from Decision Manager. Based on these metrics, orders are either accepted in Mozu or marked as needing further review.

Application Features:

  • Automatically sends information about submitted orders to Decision Manager, including the shopper's browser type, IP address, device type, billing information, etc.
  • Validates orders against your Decision Manager rules to determine a fraud score and validation result (accept, review, decline).
  • Updates order status in Mozu to Accepted, Pending Review, or Cancelled based on Decision Manager validation results.
  • Syncs order status information with Decison Manager at scheduled intervals.
  • Avoids excessive overhead for low-value orders by allowing you to set a monetary threshold below which orders are not screened for fraud.
  • Maps Mozu data to Decision Manager to facilitate the creation of fraud detection rules based on fields such as a shopper's email address or payment type.

Install the Mozu CyberSource Decision Manager Application on Your Tenant

After purchasing the Mozu CyberSource Decision Manager Application, call your sales or professional services representative and request installation on your tenant. Mozu staff will install the app to your tenant, making it available for further configuration and usage in your storefront.

Configure the Application

To integrate your Decision Manager services with Mozu, review the configuration requirements to ensure you have everything you need to be successful before you begin, and then complete the steps in this section.

Configuration Requirements

  • The Mozu CyberSource Decision Manager Application must be installed on your tenant.
  • You must have an active Decision Manager account.

Obtain Decision Manager Account Information

Note your Decision Manager account credentials. You will enter these credentials in Mozu.

  1. Log in to the CyberSource Business Center.
  2. Note the username and password that you use to log in to the CyberSource Business Center.
  3. Note your Account ID and Merchant ID.
  4. Create and note a Transaction Security Key for the SOAP Toolkit API.

    Note: If you have already created this key but lost or forgotten it, you will have to generate a new key.

Configure the Mozu CyberSource Decision Manager Application

After obtaining your Decision Manager account information, enter the information in Mozu and configure the settings for the Mozu CyberSource Decision Manager Application:

  1. Log in to Mozu Admin.
  2. On the Mozu Admin top bar, select Settings > Applications.
  3. Click CyberSource Decision Manager.
  4. Click the Configuration link to open configuration settings.
  5. Open the Settings tab.
  6. Add your CyberSource account information to the Organization ID, Merchant ID, and Transaction Key.
  7. (Optional) Enable Threshold Enabled to set a Threshold Value below or at which orders are not screened by Decision Manager. For example, you may want Decision Manager to only screen orders over $10 for fraud.
  8. Under Select Payment Type, select the payment types you want to screen with Decision Manager.

    Note: If you select multiple payment types, Mozu sends only the largest payment type to Decision Manager for screening. For example, if you set an order threshold of $100, and an order of $150 is charged as $90 via credit card and $60 via Paypal Express, only the $90 credit card payment is sent to Decision Manager for screening.

  9. Choose which Environment the Mozu CyberSource Decision Manager Application screens orders in. If you select the Test environment, the application sends order information to Decision Manager's test site, where you can evaluate fraud detection rules in a sandbox setting. When you are satisfied with the rules you set in Decision Manager, choose the Production environment to enable live fraud detection on your site.


  10. (Optional) Set an Order Synch Frequency to specify how often Mozu queries Decision Manager for updates on orders. If you accept or reject an order in Decision Manager that is Pending Review in Mozu, the order synch frequency determines how long it takes for the new order status to update in Mozu. If this option is Disabled, Mozu does not query Decision Manager, and you must manually process the order in Mozu.

    Tip: Balance the frequency at which you receive updates in Mozu with the amount of bandwidth you are comfortable using to send order sync queries from Mozu to Decision Manager.

  11. If you specify an order synch frequency, enter the User Name and Password that you use to log into the CyberSource Business Center.

  12. Click Save.

Enable the App

After configuration, enable the Mozu CyberSource Decision Manager Application in Mozu to apply its functionality to your tenant.

  1. On the Mozu Admin top bar, select Settings > Applications.
  2. Click CyberSource Decision Manager Application.
  3. Click Enable App.

Congratulations! You can now preview Decision Manager functionality on your site.

Handling Orders Flagged for Further Review

When Decision Manager screens orders it receives from Mozu, it attaches one of the following validation results to the order: accept, review, or reject. If you enabled synching by setting an Order Synch Frequency on the app configuration page, the validation result has the following impact on the order status in Mozu:

Decision Manager Validation Result Mozu Order Status
accept Accepted
review Pending Review
reject Cancelled

If you do not enable synching, all orders sent to Decision Manager remain in Pending Review until you manually process them in Mozu Admin. Complete the following steps to process an order marked Pending Review in Mozu Admin:

  1. In Mozu Admin, open the Orders module.
  2. Locate the order and click the row to open the Orders editor.
  3. Under Attributes, use the Decision Manager Fraud Results to review the reasons the order is pending review.
    1. Note whether Decision Manager accepted, rejected, or marked the order for further review.
    2. Consider fraud risk information such as the fraud score result (afsResult), risk factor codes (afsFactorCode), and reason code (ReasonCode). For help interpreting all the values listed in the Decision Manager results, refer to the Decision Manager Developer Guide: Using the Simple Order API available through the CyberSource Business Center.

      Example: In the preceding screenshot, you can determine that Decision Manager rejected the order due to a fraud score result that may be above your ignore threshold based on risk factor codes that include phone inconsistencies and high account usage.

  4. If you think the order is fraudulent, click Cancel Order. Otherwise, click Accept Order.

You can also use the Decision Manager website to accept or cancel orders pending review. If you update an order through the Decision Manager website, you see the update in Mozu in accordance with the order synchronization interval you set in the configuration settings.

Map Mozu Fields to Custom Fraud Detection Rules

If you want to add to the rules that Decision Manager uses to screen an order for fraud risk, use the Decision Manager website to modify the existing rules or create new custom rules. To help you build custom rules, you can map Mozu fields, such as email address and payment types, to available merchant-defined data fields in Decision Manager. Afterwards, you can associate the MDD fielsd with custom fields in Decision Manager and use the custom fields in your fraud detection rules. To map these fields:

  1. Log in to Mozu Admin.
  2. On the Mozu Admin top bar, select Settings > Applications.
  3. Click CyberSource Decision Manager.
  4. Click the Configuration link to open configuration settings.
  5. Open the Custom Mapping tab.
  6. Select the Merchant ID that corresponds to the CyberSource account you want to set up the mappings for.
  7. Click Add Custom Mapping.

    1. Using the Domain drop-down menu, select the type of Mozu data you want to map to Decision Manager.
    2. Select the Mozu Data that you want to map to Decision Manager. If the Mozu data is already mapped, Mozu does not display it in the drop-down.
    3. Select which merchant-defined data field you want to map the data to. If the MDD field is already in use, Mozu does not display it in the drop-down.
    4. Click Save.
  8. After you create a mapping, log in to your Decision Manager account and associate a custom field with a mapped MDD field. You create this association in order to use the custom field within a rule (because rules don't access MDD fields directly). For example, in Mozu you may have mapped a shopper's email address to MDD 7. If this is the case, you should map a custom field in Decision Manager to MDD 7 and name it "Email". In this way, the data you map from Mozu is accessible for use in Decision Manager rules via the custom field.
  9. After you complete the previous step, create custom rules in Decision Manager that utilize the custom fields you set up, thereby taking advantage of the data Mozu maps to MDD fields with each screened order.

For more information on creating custom rules and fields in Decision Manager, refer to the Decision Manager User Guide available through the CyberSource Business Center.
