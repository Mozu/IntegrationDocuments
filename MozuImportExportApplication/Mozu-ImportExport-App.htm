Welcome to the Mozu Import-Export Application

The Mozu platform collects a wide array of real-time information about your site useful for IT analytics, including error incidents, performance metrics, and unavailable links. You can use the Mozu Import-Export Application to export this information to an Amazon Simple Storage Service (S3) buckets or to an FTP or FTPS server.

Application Features:

  • Collect real-time information about your site(s) that helps you monitor errors, warnings, performance metrics, unavailable links, and uptime issues.
  • Export logged data to your Amazon S3 account for storage and analysis.
  • Export logged data to an FTP server of your choice.

Install the Mozu Import-Export Application on Your Tenant

After purchasing the Mozu-Import Export Application, call your sales or professional services representative and request installation on your tenant. Mozu staff will install the app to your tenant, making it available for configuration and usage in your storefront.

Configure the Application

To integrate the functionality of the Mozu Import-Export Application with your site, review the configuration requirements to ensure you have everything you need to be successful before you begin, and then complete the steps in this section.

Configuration Requirements

  • The Mozu Import-Export Application must be installed on your tenant.
  • You must have an Amazon S3 account or access to an FTP server if you want to export logged data from your site.

    If you use an Amazon S3 account, ensure your account has a bucket in which to store files. The Mozu Import-Export Application sends the logs from your site to a bucket on your S3 account. You can create a bucket using the S3 Console in Amazon Web Services.

    If you use an FTP server, ensure you have access to a directory in which to store the logs sent from the Mozu Import-Export Application. Your IT department or FTP administrator can help identify a suitable directory.

(Optional) Obtain Amazon S3 Security Credentials

If you want to export logs to an Amazon S3 account, note the security credentials for the account:

  1. Log in to Amazon Web Services.
  2. Navigate to the Security Credentials page under your account drop-down.
  3. Note your Access Key ID.

    Note: You also need to know your secret access key. However, you cannot view the secret access key on your Amazon account after the initial creation process. If you have lost or forgotten your secret access key, you need to create a new set of access keys. You can create new access keys for the account by going to the Security Credentials page. In the Access Keys section, click Create New Access Key.

  4. Navigate to the Account Settings page.
  5. Under Security Token Service Regions, note your active regions.

(Optional) Obtain FTP Server Credentials

If you want to export logs to an FTP server, note the following details about your FTP setup. If you need help obtaining these details, contact your IT department or FTP administrator.

  • Server name
  • Username
  • Password
  • Whether the server uses Secure FTP
  • Port number
  • A directory on the server in which to store the logs of exported data

Configure the Mozu Import-Export Settings

  1. On the Mozu Admin top bar, select Settings»Applications.
  2. Click Mozu Import-Export Application.
  3. Click the Configuration link to open configuration settings.
  4. Open the Options tab.
  5. Configure the options for the Mozu Import-Export Application. You can enter configuration information for both an Amazon S3 account and an FTP server.
    1. If you want to export data to an Amazon S3 account, enable Enable S3 Account.
    2. Enter your access key ID, secret key, and the service region for your Amazon S3 account.
    3. If you want to export data to an FTP server, enable Enable FTP.
    4. Enter the server name, username, and password for the FTP server.
    5. Enable Use Secure FTP if the FTP server uses SFTP.
    6. Enter the port number for the FTP server.
  6. Click Save.

Enable the App

You do not need to complete additional steps to enable the Mozu Import-Export Application. You can disregard the Enable App button.

Use the App

  • Export Logs

Export Logs to an Amazon S3 Account or to an FTP Server

Use the configuration page of the Mozu Import-Export Application to schedule exports of IT data about your site. You can also view the log history of scheduled exports using the configuration page.

Schedule an Export

  1. On the Mozu Admin top bar, select Settings»Applications.
  2. Click Mozu Import-Export Application.
  3. Click the Configuration link to open the configuration settings.
  4. Click Web Logs.
  5. Click Show Export Config to configure export settings.
  6. Using the Select External Storage drop-down, choose to export data to Amazon S3 or to an FTP server. Note that you cannot export data to both destinations simultaneously. If you want to switch between destinations, you will have to stop an export and configure a new one.
  7. Using the Export Interval (minutes) drop-down, choose how frequently to export data.
  8. (Amazon S3) Using the S3 Bucket drop-down, choose an Amazon S3 bucket to receive the logs in.
  9. (FTP) In FTP Directory, enter the directory on the FTP server to receive the logs.
  10. Click Save.

Note: If you want to change these settings after configuring an export, stop the export first and then make the changes.

View Logs and Export History

To view full log information, access your Amazon S3 account or FTP server and open the logs.

To check what exports the Mozu Import-Export Application has generated, use the Web Logs page to view a history of exported logs. You can review the date and time of a log and verify whether the export completed successfully or not. If you want to view additional information about an export, click View in the appropriate row.

Stop an Export

You can stop an export at any time:

  1. Open the Web Logs page.
  2. Click Stop Export.
